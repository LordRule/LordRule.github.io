一、Linux中配置Anaconda环境&& 利用Nginx + uWSGI部署Django项目（内网服务器）

1.1 概述

在内网环境中部署大数据训练模型项目，选择使用Anaconda虚拟环境管理项目，模型项目由Django框架编写，选择Nginx + uWSGI进行部署。在本地环境中将Django项目编写完成，要将项目部署到服务器中成功运行，就需要保证服务器的运行环境和本地环境一致。本地环境系统是Windows，服务器系统CentOS7，环境不相同就不能直接在本地导出aconda虚拟环境，在服务器中加载。此时，可以考虑两种方案：

方案一：导出本地conda环境依赖包配置目录（requirements.txt）,寻找能够连接外网的centOS7服务器（保证内网和外网的服务器版本和配置差不多类似），在外网服务器中按照requirements.txt下载保存环境依赖包。最后将依赖包利用FTP软件上传至内网服务器中，在服务器中安装配置对应环境。

方案二：考虑使用Docker容器化技术来部署，在外网环境中下载好Docker镜像内容和对应依赖包，将镜像文件导入到内网环境中（内网环境也需要安装对应版本的Docker）。在本地Django项目目录下创建Dockerfile文件，写入具体配置信息。（读者感兴趣的话，可以自行百度）

Docker容器技术之前有所涉及，秉着学习新知识和新技术想法，选择方案一，在服务器中搭建Anaconda虚拟环境管理项目，在虚拟环境中部署Django项目，使用Nginx进行外部网络代理服务器访问和uWSGI内网Web服务器配置。下面将详细介绍各个模块的作用：

WSGI是一种python专用的web协议（和http类似），WSGI是一种规范，定义使用python编写的web app（Django）与web server（uWSGI）之间接口格式，实现web app 和 web server之间的解耦。

uWSGI是一个全功能HTTP的服务器，实现WSGI协议、uwsgi协议、http协议等。主要作用是把http协议转化为WSGI协议，让Python可以进行直接使用。特点为轻量级、易部署，性能比Nginx差很多。

注： 架构为Nginx+uWSGI+ APP，uWSGI为一个中间件；架构为uWSGI+APP，uWSGI为一个服务器。

Nginx是一个web服务器，多用于反向代理、负载均衡、拦截攻击等，而且性能极高。

Django是一个python的web框架，框架的作用是处理request和response，所以如何部署Django，Django本身并不关心。

请求处理整体流程：

  • nginx接收到浏览器发送过来的http请求，将包进行解析，分析url
  • 静态文件请求：就直接访问用户给nginx配置的静态文件目录，直接返回用户请求的静态文件
  • 动态接口请求：那么nginx就将请求转发给uWSGI，最后到达django处理

各模块作用：

  • nginx：是对外的服务器，外部浏览器通过url访问nginx，nginx主要处理静态请求
  • uWSGI：是对内的服务器，主要用来处理动态请求
  • uwsgi：是一种web协议，接收到请求之后将包进行处理，处理成wsgi可以接受的格式，并发给wsgi
  • wsgi：是python专用的web协议，根据请求调用应用程序（django）的某个文件，某个文件的某个函数
  • django：是真正干活的，查询数据等资源，把处理的结果再次返回给WSGI， WSGI 将返回值进行打包，打包成uwsgi能够接收的格式
  • uwsgi接收wsgi发送的请求，并转发给nginx,nginx最终将返回值返回给浏览器
